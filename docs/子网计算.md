# NETKit IP子网计算工具设计文档

## 项目背景

NETKit是一个网络工具集，目前已包含IP配置和Ping测试功能。用户希望添加IP子网计算工具，这是网络管理员和网络工程师经常使用的重要功能。

## 需求分析

### 用户需求
- 添加专业的IP子网计算功能
- 与现有功能保持一致的用户体验
- 提供完整的子网计算和分析能力

### 功能对标
参考常见的IP子网计算工具：
- 在线工具：subnet-calculator.com、calculator.net等
- 桌面软件：Advanced IP Scanner、SolarWinds IP Address Manager等
- 命令行工具：ipcalc、sipcalc等

## 功能设计

### 核心功能模块

#### 1. 基础子网计算器
- **输入方式**：
  - IP地址 + 子网掩码（如：192.168.1.0 + 255.255.255.0）
  - CIDR表示法（如：192.168.1.0/24）
- **计算结果**：
  - 网络地址、广播地址
  - 子网掩码（十进制和二进制）
  - 可用主机数量
  - 主机地址范围（第一个可用IP到最后一个可用IP）
  - 网络类别（A/B/C类）
  - 是否为私有地址

#### 2. 子网划分工具
- **按主机数划分**：输入需要的主机数，自动计算最小子网掩码
- **按子网数划分**：输入需要的子网数量，计算如何划分
- **自定义划分**：手动指定多个子网的大小需求

#### 3. VLSM计算器
- 输入不同部门的主机需求
- 自动优化子网分配，减少地址浪费
- 显示详细的分配方案

#### 4. 地址范围工具
- **地址包含检查**：检查某个IP是否在指定子网内
- **子网重叠检查**：检查两个子网是否有重叠
- **地址汇总**：将多个连续子网汇总为超网

#### 5. 实用工具
- **二进制转换器**：IP地址和子网掩码的二进制显示
- **地址类型识别**：公有/私有、单播/组播/广播
- **反向计算**：根据主机范围反推子网信息

### 详细界面元素

#### 1. 输入区域
- **IP地址输入框**: 支持网络地址输入，带占位符提示
- **CIDR下拉框**: /8到/30的常用选项，也可手动输入
- **子网掩码输入框**: 传统格式输入，与CIDR联动
- **按钮**: 使用现有的蓝色主题按钮样式

#### 2. 基础结果显示区
- **信息展示**: 使用类似"当前网卡信息"的GroupBox样式
- **字体**: 保持与现有界面一致的Microsoft YaHei UI
- **颜色**: 使用Constants中定义的文字颜色

#### 3. 高级功能Tab控件

**Tab 1: 基础信息**
- 二进制显示
- 地址类型信息
- 网络类别说明

**Tab 2: 子网划分**
- 划分参数输入
- 划分结果展示

**Tab 3: VLSM计算**
- 部门需求输入
- 优化分配方案

**Tab 4: 地址工具**
- 地址包含检查
- 子网重叠检查

## 技术实现

```

│   │   └── VLSMRequirement.cs            # VLSM需求模型

    ├── Controls/
    │   ├── SubnetCalculatorPanel.cs       # 子网计算主面板
    │   └── SubnetResultDisplayControl.cs  # 结果显示控件
    └── Forms/
        ├── MainForm.cs (修改)             # 添加新Tab
```

### 核心算法

#### 基础计算
- IP地址与整数的相互转换
- 子网掩码的位运算
- 网络地址和广播地址计算
- 主机数量计算（2^(32-前缀长度) - 2）

#### 数据验证
- IP地址格式验证
- 子网掩码有效性检查
- CIDR范围验证（/0 到 /32）

### 用户体验设计

#### 交互特性
- **实时计算**: 输入变化时自动更新结果
- **智能提示**: 输入错误时显示红色提示
- **一键复制**: 点击结果可复制到剪贴板
- **历史记录**: 保存最近计算的网段

#### 设计原则
1. **一致性**: 完全遵循现有的UI风格和颜色方案
2. **实用性**: 重点突出最常用的功能
3. **层次性**: 通过GroupBox和Tab组织信息层次
4. **响应式**: 输入变化时实时更新计算结果
5. **专业性**: 提供网络工程师需要的详细信息
